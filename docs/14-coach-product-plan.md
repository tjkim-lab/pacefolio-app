# 14. 코치 화면 제품 원칙 + 배치 C1~C5 계획 (코치 관점 피드백, 2026-07-18)

> 최상위 원칙(확정): **코치 앱은 관리 대시보드가 아니라 현장 도구** —
> 오늘 수업 확인 → 출석 체크 → 프로그램 완료 → 필요한 기록·안내. 끝.
> 핵심 3가지: ① 오늘 어떤 수업이 있는지 ② 준비→시작→출석·진행→완료 ③ 필수 공지 확인.

## 구현 상태 표기 규칙
- **UI_ONLY**: 구동 목업으로 구현 — fixture/로컬 상태, API 없음. 완료 처리 아님.
- **API_REQUIRED**: 백엔드 필요 — 화면에 명시적으로 표기해 둠.
- 출석·수업 완료·사고 기록·공지 확인 = **감사 로그 필수 업무 이벤트** (배치 14/출결 API 에서
  AuditLog·서버 상태·멱등키·테넌트/담당 경계 검증과 함께 구현).

## C1 — 코치 홈 MVP ✅ UI_ONLY 완료 (2026-07-18)
- 오늘의 수업 최상단 · 시간순 캐러셀(공용 HomeBanner 144px 규격) · 인디케이터
- 카드: 시작–종료 시간 · 반 이름 · 학생 수 · 장소 · 상태 배지 · **상태별 단일 액션 버튼**
- 상태 모델 `SCHEDULED→READY→IN_PROGRESS→COMPLETED(+CANCELLED)` — 목업은 코치
  상태머신 파생, **정본 = 서버 수업 상태(API_REQUIRED)** · 완료 중복 클릭 멱등 가드
- 필수 확인 공지: 확인 버튼 전까지 유지(READ ≠ ACKNOWLEDGED) · 확인 시각 원장 표시
- 학부모 전달사항: 원생+수업 컨텍스트 표시(fixture 예정결석 파생)
- 위젯 최소화: "다음 수업 확정" 기본 접힘

## C2 — 수업 모드 ✅ UI_ONLY 완료
- 출석 4상태(출석·결석·지각·조퇴) 한 탭 순환 · 미지정 시 저장 불가 경고 · 변경 이력
- 프로그램 = **영역 — 활동명 + 완료만** (분·초·횟수·측정값 제거 — 프로그램 정책 확정 후 후속)
- 특이사항·안전사고 한 줄 액션: `[⚠ 기록] [기록 후 원장 알림]` (줄바꿈 금지 규격)
  — 안전사고는 원장 즉시 알림 + 원장 홈 확인 필요 영역 노출 + 확인까지 유지(배치 14 연동)
- 발송(완료) 멱등: sending/reportSent 중복 실행 차단

## C3 — 기록·메시지 ✅ UI_ONLY 완료 (학부모 시각화 잔여)
- 완료 활동 → 원생별 **성장 영역**(균형감각·협응성·인지·집중, 확장 가능) 누적 표시
  — "능력 진단·점수 아님, 활동 경험 기록" 명시. 영역↔프로그램 연결은 추후 변경 가능 설계
- 공통 메시지 템플릿 최대 5: 생성(+현재 문구 저장)·삭제(내 정보›설정)·선택 후 수정 ·
  치환 변수 설계 `{학생이름}{수업명}{수업날짜}{오늘활동}{코치이름}` (치환 실행 = API_REQUIRED)
- 개별 학생 피드백(공통 메시지와 분리): 내 정보 › 담당 수업 › 학생 → 공개 범위
  (코치 내부 메모 / 원장과 공유 / 학부모에게 전달) 명시 후 작성
- **잔여**: 학부모 앱 주간 영역별 채움 카드·4주 분포 시각화 — 다음 배치

## C4 — 내 정보·설정 ✅ UI_ONLY 완료
- 순서: 이번 주 일정(오늘 강조) → 담당 수업(학생 목록+피드백) → 코치 카드·근무 학원 → 코치 설정
- 코치 설정: 내 인사정보(이름·연락처·입사일·담당·자격증 유효기간·응급처치·계약 형태 —
  급여·계약서는 원장 권한+열람 로그) · 템플릿 관리 · 알림 · **인수인계(고정 메뉴→설정 안)** ·
  사진·미디어 · 계정·보안
- 명칭 통일: 사용자 노출 "채팅/코치방/원장방" → **소통** (코치·학부모 탭 완료 · 내부 방 유형은 구분 유지)

## C5 — 사진 인프라 (후속 배치 · 코드 착수 전 정책 확정)
- **공개 URL 금지** — 오브젝트 스토리지(S3/R2/GCS/… 교체 가능한 인터페이스) + 만료 signed URL
- 흐름: 코치 앱 → 업로드 권한 발급 → 스토리지 직접 업로드 → 서버엔 메타데이터만 →
  보호자는 권한 있는 사진만 다운로드
- 필수: 압축·원본/썸네일 분리·크기/개수 제한·학원/수업/원생 접근 권한·보관 기간+자동 삭제·
  삭제 요청·감사 로그·미성년자 동의 확인·푸시에 사진/민감정보 미노출
- 제품 흐름: 업로드 → "오늘 수업 사진 등록" 알림 → 앱에서 열람/다운로드 · 보관 종료일 표시 ·
  기간 내 일괄 다운로드. 외부 웹하드 링크 방식은 배제(권한 회수·원생별 통제 불가)

## 후속 검토 (우선 3: 학생별 주의사항 빠른 확인 · 대체 수업/코치 변경 · 오프라인 임시 저장)
휴가·대체 요청 / 수업 변경 / 자격증 만료 알림 / 준비물 요청 / 수업 전 체크리스트 /
미확인 공지 모아보기 / 완료 전 누락 자동 확인 / 알림 시간 설정 / 비상 연락 절차

## API_REQUIRED 백로그 (배치 14·출결 API 와 연결)
서버 수업 상태·전이 / 출석 저장(감사) / 활동 영역 누적 저장 / 안전사고 → 원장 확인 필요 큐 /
ACK 메시지 상태머신 / 템플릿 저장 / 개별 피드백 저장(공개 범위 강제) / 담당·테넌트 경계 서버 검증
