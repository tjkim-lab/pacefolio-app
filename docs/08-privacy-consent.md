# 08 · 개인정보 · 사진 동의 (F12)

코드: `lib/domain/consent.ts`. 리뷰 P0-7 · 4-1 · 4-2. 권한은 docs/04와 결합.

## 1. 사진 동의 (리뷰 P0-7)
`photoChecked: boolean` **금지.** 목적·대상·버전·만료·철회를 가진 모델.
- **목적**: 개별전달 · 반공유 · 내부기록 · 학원홍보 · 외부광고 · SNS.
- **대상**: 보호자만 · 반구성원 · 학원내부 · 공개.
- **재검증(`canSendPhoto`)**: 발송·범위확대·철회 확인을 **매 발송 시점** 서버가.
  - 철회됨 → 차단 / 만료 → 차단 / 목적·대상 미포함 → 차단.
- 범위 확대(내부→홍보)는 **재동의 필요**. 코치가 사진 검토 후 공개범위 넓히면 재검증.

## 2. 민감필드 공개범위 (리뷰 4-2)
> **SNS식 임의 "Everyone" 금지.** 시스템 정책(`PrivacyVisibility`)으로 고정, 사용자가 확대 불가(`isUserOverridable`=false).

| 필드 | 고정 공개범위 |
|---|---|
| 코치·보호자 전화 원문 | 원장·승인 관리자 (그 외 중계전화·인앱채팅) |
| 원생 생년월일 | 연결 보호자·승인 담당자 |
| 원생 건강·안전정보 | 담당/대체 코치·원장·안전담당 최소인원 (+조회기록) |
| 결제 금액 | 보호자(본인)·원장·수납담당 |
| 사진 동의 | 보호자·학원 관리자 |

## 3. 원생 안전정보 (리뷰 4-1)
의료·건강정보는 **일반 프로필이 아니라 원생 안전정보로 분리**: 알레르기·복용약·응급조치·활동제한·보호자 긴급연락·의료기관.
- 접근: 보호자 · 담당/대체 코치 · 원장/승인 안전담당. **다른 보호자·다른 반 코치·데스크·홍보·외부 접근 불가.**
- **최소 수집** + **조회 기록(AuditLog)**.
