- README 드리프트: 학부모 청구·결제 실 API 모드 반영(상태 정본 우선순위 준수)
- 학부모 성장 화면의 측정형 표현("한발 서기 15초·18초 신기록") 제거 —
  제품 원칙("점수·측정 아님 — 활동 경험") 전 화면 정합(코치 화면은 C2 에서 기제거)
- 도메인 계약 선확정: `CLASS_SCHEDULE_TYPE` 3종 · `PG_ONBOARDING_STATUS` ·
  `PAYMENT_CHANNEL`(온라인/이체/현금/현장카드 — 수기 상태 변경 수납 금지)

## 경쟁사에서 가져오는 것 / 안 가져오는 것
**가져옴**: 수업 유형 3종 · 공지→행사 캘린더 자동 등록(#22 일정과 연결) ·
PG 온보딩 상태 · 오프라인 수납(단, 별도 채널 기록) · 강사 초대 수명주기(기존
MEMBERSHIP_STATUS 의 INVITED 활용) · 반별 월간 통계(원천 API 완성 후)

**안 가져옴(반면교사)**: 코치에게 매출·수납·청구 권한(→ 역할 분리 유지) ·
쪽지함형 알림장(→ 대화 thread·ACK 모델 유지) · 화면에서 수기 상태 변경 수납
(→ 웹훅/수납 이벤트 분리) · 모바일 전기능 탑재(→ 모바일=확인·승인/PC=작성·설정)

## 보류·준비물 (코드 밖)

강의별 게시판·댓글은 소통 모델(thread)과 중복 — 학원방 공지형으로 흡수 검토.
