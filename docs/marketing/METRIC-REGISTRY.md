# 지표 사전 (마케팅 리뷰 A-5·A-6·A-7)

모든 지표는 `metricId · 분자 · 분모 · 기간 · timezone(Asia/Seoul) · 포함/제외 · 데이터 지연 · 최소 표본 · 재계산 정책 · 추정치 여부 · 공개 범위`를 갖는다. 산식 변경 = 버전 업.

## NSM-1 자동결제 등록률 (북극성) — v1
- **분자**: 월말 기준 활성 자동결제 수단이 있는 **결제 가능 가구**(billing account) 수
- **분모**: 자동결제 대상이며 활성 청구 관계가 있는 전체 가구 수
- **단위 = 가구**(User 아님 — 중복 보호자 왜곡 방지, A-6)
- **제외**: 자동결제 불가 PG 수단 · 탈퇴/퇴원 가구 · 테스트 계정 · 무료 수강 · 결제 권한 없는 보호자
- 기간: 월말 스냅샷 / 원천: `autopay_registered`(pg) + 청구 관계(서버) / 최소 표본: 분모 5가구 미만이면 비율 숨김

## NSM-2 수납 가시성 사용률 — v1
- **분자**: 월간 수납 업무가 존재한 학원 중, `collection_dashboard_action`(서버 기록 유효 행동: 수납현황 확인·미납 필터·리마인드 발송·정산 내보내기) **1회 이상** 학원 수
- **분모**: 해당 월 수납 업무(발행 청구서 ≥1)가 존재한 학원 수
- **제외**: 페이지뷰만 있는 경우 · 백그라운드 로딩 · 자동 refresh (A-6 — 행동 기준)

## EST-1 절감 시간 (추정치 — 실측 아님, A-7)
- **측정값**(그대로 표시): 자동 안내 발송 N건 · 자동 정산 완료 N건 · 앱 처리 결석통보 N건
- **추정값**(추정임을 UI 에 명시): `Σ(행동 수 × assumedMinutesPerAction)`
- UI 문구: *"앱에서 처리한 업무를 기준으로 약 N분을 아끼신 것으로 **추정돼요**"* — "전화가 줄었다" 단정 금지(외부 전화기록 없음)
- 버전 필드: `estimateModelVersion · assumedMinutesPerAction · calculatedAt · sourceMetricIds`
- v1 가정: 결석통보 5분/건 · 수납 리마인드 3분/건 · 정산 10분/건 (검증 후 조정)

## 공통
- 지표별 owner/HQ 공개 범위 분리(테넌트 projection — GROWTH-SNAPSHOT-SPEC)
- 소수 표본 억제: 분모 < 5 → 비율 숨기고 "재등록 대상 1명 중 1명" 식 원자료 표기(A-14)
- 데이터 지연 표기: pg 지표 D+1 · server 지표 실시간 · client 지표 참고용
