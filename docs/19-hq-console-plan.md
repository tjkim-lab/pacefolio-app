# 19. 본부(HQ) 콘솔 계획 — 다락팩토리 도구의 PACEFOLIO 콘솔 이관

> 2026-07-19 TJ 결정: **PACEFOLIO 콘솔(admin) = 다락팩토리 본부.**
> `dev/crawler-tool`(FastAPI·SQLite·NCP VM·crawler.wondergym.cloud)의 기능을 콘솔로 이관.
> 원칙: **크롤러 저장소는 불변(읽기 전용) — 필요한 것만 가져온다.** 인벤토리는 실코드 전수 조사
> (파이썬 257파일 / 약 50K LOC / 서비스 108 / 라우터 48 / 크롤러 어댑터 ~30 / 크론 잡 ~20) 기준.

## B. 콘솔 본부 메뉴 매핑 (헌법 정합)

| 본부 메뉴(신설/확장) | 가져올 기능 | 헌법·전략 근거 |
|---|---|---|
| **커머스 직영** | 상품관리·등록 준비·가격/재고 추적·커머스API 대조 | "커머스 = 페이스폴리오 직영" |
| **콘텐츠 스튜디오** | 블로그 팩토리·매거진 발행 파이프라인·유튜브 도구 | 매거진(/magazine 한 트리)·마케팅 3엔진 |
| **네이버 관제탑** | 검색/데이터랩·스마트스토어 연동·(향후 톡톡/플레이스) | NAVER-PRESENCE-SPEC·성장판 |
| **AI 스튜디오** | 상품명/상세/이미지 변환·비전 판별·프롬프트 라이브러리 | AI-STRATEGY("AI 초안→사람 확정") |
| **운영(기존 admin 확장)** | 알림(네이버웍스 채널 추가)·활동로그·주간 업무 리포트 | outbox 디스패처에 채널만 추가 |
| 기존 테넌트 관제 | (그대로) 구독·MRR·정지·SupportView | admin 경계 유지 |

**경계 원칙**: 본부 기능 = PLATFORM_ADMIN 표면. 테넌트(학원) 데이터와 본부(커머스·콘텐츠) 데이터는
스키마·권한에서 분리 유지. console-admin 물리 분리(B5)가 이 확장의 그릇.

## C. 이관 3분류 (조사 소견 기반)

### ① 1단계 — 연동 (콘솔이 입구, 크롤러는 무중단)
- 방식: 크롤러 FastAPI 는 **"수집·AI 엔진" 서비스로 존속**, 콘솔 본부 메뉴가 그 API 를 호출/임베드.
- 대상: 상품관리 조회·잡 트리거·AI 변환 현황·가격 추적 보드 등 "보는 것 + 버튼".
- 필요 작업: 크롤러에 서비스 간 인증(API 토큰) 1개 추가 요청(크롤러 쪽 최소 변경 — 협의),
  콘솔에 본부 메뉴 껍데기 + 프록시. **직원 워크플로 무중단.**

### ② 2단계 — 선별 이식 (PACEFOLIO 데이터와 결합 가치 큰 것부터)
stateless(요청/응답)라 이식 쉬움(조사 소견):
1. **알림 채널** — 네이버웍스 봇 발송을 outbox 디스패처의 채널로 이식(가장 작고 즉효)
2. **네이버 검색/데이터랩 래퍼** — 성장판 카드·매거진 키워드 검증에 직결
3. **콘텐츠 발행 파이프라인** — 매거진(/magazine) 발행과 직결되는 부분부터
4. AI 호출 계열(상품명·상세·비전) — Gemini 단일 벤더라 추상화 1개로 이식 가능
5. 구글 시트/드라이브 연동(커머스 직영 운영에 필요 시)

### ③ 잔류 — 수집 엔진 (이식 비용 > 가치, 결과만 API 로)
- Playwright 크롤러 어댑터 30종 + 워커/큐/SSE/크론 오케스트레이션 — 헤드리스 크롬 런타임 통째
- yt-dlp+ffmpeg 영상 다운로드(현재도 로컬 PC 전용)
- products 상태머신(컬럼 수십 개) — 이식하면 스키마 마이그레이션 대형. 미러 조회 API 로 소비

## D. 페이징 제안

| 페이즈 | 내용 | 전제 |
|---|---|---|
| HQ-0 | 본부 메뉴 IA(정보구조) 확정 + 콘솔 껍데기 — 디자인 터미널 협업 | 파일럿 P0 이후(완료됨) |
| HQ-1 | 연동: 크롤러 API 토큰 + 콘솔 프록시 + 상품관리/AI 현황 조회 | 크롤러 최소 변경 협의 |
| HQ-2 | 이식 1~2번(네이버웍스 알림 채널·검색/데이터랩 래퍼) | 없음(코드만) |
| HQ-3 | 콘텐츠 파이프라인 ↔ 매거진 결합 | pacefolio.co.kr 매거진 트리 가동 |
| HQ-4 | AI 스튜디오·커머스 직영 심화 | 커머스 직영 정책 결정 |

**우선순위 유의**: 본부 이관은 대형 트랙 — **파일럿(테넌트 제품) 준비물(카카오 키·법률·배포)과
경쟁하지 않게** HQ-1~2(저비용)만 파일럿 병행, HQ-3+ 는 파일럿 안착 후.

## E. 리스크·미결
- 크롤러 서버(3.8GB)와 PACEFOLIO 는 VM 분리 확정 — 연동은 내부망/터널 경유
- 크롤러 쪽 "최소 변경"(API 토큰)도 저장소 수정임 — **크롤러 작업은 별도 승인 후**(현 원칙: 읽기 전용)
- SQLite(크롤러) ↔ PostgreSQL(PACEFOLIO) 이중 정본 기간의 데이터 소유권: 커머스·콘텐츠 정본은
  당분간 크롤러 SQLite, 콘솔은 소비자 — 이식 완료 축부터 정본 이동

## F. 진행 기록
- **2026-07-19 HQ-1 콘솔 골격 구현(#43)**: `apps/api/src/hq/service.ts` + `/admin/hq/{health,products,jobs}`
  (PLATFORM_ADMIN 전용, env 미설정 = 501). 크롤러 실코드 조사(읽기 전용) 결과 반영:
  - 크롤러 인증 = **세션 쿠키(SessionMiddleware) + IP 화이트리스트** — 서비스 토큰 없음.
    현 골격은 `PACEFOLIO_HQ_CRAWLER_BASE_URL` + `PACEFOLIO_HQ_CRAWLER_COOKIE`(로그인 세션) env 주입.
    **서비스 토큰 전환 = 크롤러 최소 변경 필요 → 별도 승인 후.**
  - 소비 엔드포인트(전부 GET·JSON): `/api/v1/health`(인증 우회)·`/api/v1/products`(limit≤200,
    reg_status)·`/api/v1/jobs/active`·`/api/v1/jobs/recent`·`/api/v1/last-crawl-summary`.
  - 활동로그·서버점검·worker-status 는 HTML(HTMX 조각) 응답이라 프록시 부적합 — JSON 화는 크롤러 변경 필요.
  - 남은 것: TJ env 주입(내부망/터널 + 크롤러 로그인 쿠키) → 콘솔 admin 화면(HQ 메뉴 IA = 디자인 협업 HQ-0).
