# QA 시나리오 원장 v1 — 고객 생애주기 162 시나리오 판정

> 기준: 시나리오 트리 v1 (2026-07-17 수신) × 코드베이스(환불 P0 트랙 마감 시점).
> 판정값: PASS-AUTO / PASS-MANUAL / FAIL / BLOCKED / NOT-IMPLEMENTED / NOT-TESTED.
> P0 는 "화면이 보인다"로 통과 처리하지 않음 — API·권한·DB·비동기·감사까지.
> 이 문서는 판정 요약. 시나리오별 10칸 상세 시트는 후속(테스트 파일이 1차 증거).
> LCV1 검토 반영: 총 162개(120 표기 정정) · 최종판정 + Layer Coverage(UI/API/AUTHZ/DB/
> ASYNC/NOTIFICATION/AUDIT/RECONCILE/RECOVERY) 2축 — "백엔드 계층 PASS ≠ 생애주기 PASS".
> LCV1-P0-01·02(웹 경계·noindex)=proxy.ts+metadata 로 mock 단계 조치 완료(실 세션 검사는
> Gate 2), P0-03(승인 링크 재검증)·P0-04(생년)=수정 완료, P0-05~07(금액검증·RECONCILE
> worker·Outbox publisher)=실 provider 트랙(Gate 3~4).
