# 프로그램 상품화 준비 — 설계 (PS7 · docs/20 §6)

> 1차 구현 = **결제형 마켓플레이스 없음**(지시서 §11). 이 문서는 데이터 모델이
> 상품화를 막지 않도록 개념·경계만 확정한다. 상태: 2026-07-21 설계 · 판매 미구현.

## 1. 소유권 모델 (스키마에 이미 확보)

| ownershipType | 의미 |
|---|---|
| `PRIVATE_ACADEMY` | 학원이 만든 사유 프로그램 — **원더짐 초기값**(visibility PRIVATE) |
| `PLATFORM_TEMPLATE` | 본사가 제공하는 시작 템플릿(복사해서 시작) |
| `MARKETPLACE_PRODUCT` | 판매용 원본(판매자 학원/본사 소유) |
| `INSTALLED_COPY` | 구매 학원에 설치된 **복사본** |

## 2. 불변 원칙 (설치 = 복사, 공유 아님)

- 상품 설치 시 **구매 학원 테넌트에 복사본 생성** — 원본 프로그램과 운영 데이터 분리.
- **구매 학원의 원생·수업·성장 데이터는 원본(판매자) 테넌트로 절대 흐르지 않는다.**
- 설치본은 구매 학원이 자유 수정(자기 활동 라이브러리로 재연결·현지화).
- 원본 개정판 업데이트는 자동 반영 안 함 — "새 버전 설치" 명시 행위로만(과거 수업 불변 원칙과 동일).

## 3. 구현된 선행 조각 (학원 내)

- `duplicateProgram` API — 프로그램 전체(모드·레벨·커리큘럼·기술·기준) 복사 → 새 프로그램 + DRAFT v1.
  활동은 **학원 공유 자산이라 개정판 참조 유지**(복사 아님). 뱃지 정의는 운영물이라 복사 제외.
- 이 함수가 곧 "설치본 생성"의 코어 — 교차 테넌트 설치는 여기에 **활동 복사 단계**를 추가하면 됨
  (타 학원 활동 참조 금지 → 설치 시 활동+개정판까지 딥카피).

## 4. Export/Import 패키지 (후속 설계 초안)

```
package.json(스키마 버전·서명) + program.json + levels[] + activities[](revisions 포함)
+ growth-domain-names[](이름 매핑 — 설치 학원 영역과 이름 매칭·미존재 시 생성 제안)
+ curriculum[] + skills[](criteria 포함)
```
- import 는 **PS3 스테이징 원칙 재사용**: 미리보기 전 운영 무변경·중복 후보·원장 확인 후 커밋.

## 5. 하지 않는 것 (1차)

결제·정산 · 마켓 노출·검색 · 라이선스 시트 수 · 원본-설치본 동기화. 전부 모델이 막지 않음만 확인.
