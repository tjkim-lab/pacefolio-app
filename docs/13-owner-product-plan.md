## A. 공용 모바일 배너 (학부모·코치·원장 규격 통일)
- full width(좌우 여백 제외) · 높이 144px 전후 · 원본 1200×500(≈2.4:1) · radius 16~18px
- 좌우 swipe + 하단 indicator 동일 · 텍스트 안전영역 통일 · 최대 3~5개
- 유형: 서비스 광고 · 중요 공지 · 시즌 이벤트 · 기능 안내 · 역할별 운영 요약(숫자만 역할별,
  외형 규격 동일) — 관리자에서 등록(어드민 마케팅 모듈과 연결)

## B. 원장 홈 (apps/web/app/owner/page.tsx)
- **수납 카드**: "기한 초과 5명 · ₩1.6M" → **₩1.6M 제거**(명단 연 후 필요할 때만 표시).
  세로 padding 축소, 요약 포함 144px 내, 숫자 타일 48~52px. 핵심 = "81/7/5" 세 숫자 = 전부 버튼
  - 결제 완료 81명 → 명단(원생·반·결제일·수단·청구액) + 결제 상세/영수증/대화
  - 결제 대기 7명 → sheet 명단(보호자·마감일·열람 여부·마지막 알림) + 알림/대화/청구서/선택 일괄 알림
  - 기한 초과 5명 → 명단(초과 일수·청구액·마지막 리마인드·연락 상태) + 리마인드/대화/결제 내역/
    **계좌이체 입금 확인**(⚠️ 단순 토글 금지 — 확인 모달+AuditLog, Payment 임의 CAPTURED 금지,
    별도 수납 경로·증빙)/운영 메모
- **오늘 처리할 일**: 기본 상태 설명 숨김(행 클릭→sheet/chevron/펼치기), 행 높이 52~56px,
  제목+숫자 중심. 액션 버튼 전부 동일 규격(76~84×36px, 동일 radius/font/padding, 한 줄) —
  완료 후 같은 자리 상태 badge(완료·발송됨·확인됨)
- **반별 정원**: 기본 접힘 accordion + 상단 요일 필터(전체·월~일, 복수 선택 = **OR**,
  "월수반만" 정확 일치는 추후 옵션) + 결과 수 표시. 펼치면 프로그램·요일시간·코치·인원/정원·
  대기자·명단·빈자리. **모바일에서 제거**: 반 상세·학부모 공지 만들기·홍보 콘텐츠 만들기(PC 로)
- **코치 전달사항**: 일반 채팅 아님 — ACK_REQUIRED 기본(긴급 = URGENT_ACK_REQUIRED).
  유형·상태·감사 = `docs/12-communication.md` 개정판 참조

## C. PC 운영 flow
- **AudienceFilter 정본**: 요일(복수 OR)·시간대·프로그램·반·코치·재원 상태 —
  원생 조회·공지 대상·청구 대상·출결 명단·대회 초대·CSV 가 **같은 필터를 재사용**
  (화면마다 별도 구현 금지)
- **휴무 → 회차 계산**: 휴무 event 등록(유형: 공휴일·방학·공사·행사·강사 개인 사유·대회·임시·기타 /
  날짜·종일/시간대·전체/특정 반·사유·회차 차감·보강 생성·수강료 차감·보호자 공지·등록자) →
  계산기가 `기본 12회 − 휴무 1회 + 추가 0회 = 11회` 도출. 숫자 직접 수정 금지 —
  수동 조정은 전/후·이유·조정자·승인자·영향 청구서 기록. 확정 청구서는 자동 변경 금지(수정 청구·차기 이월)
- **대량 청구 = 필터 기반 단계 발송**: "나머지 90명 자동" → "대상 선택 후 청구 확정".
  DRAFT→REVIEWED→CONFIRMED→SENT. 발송 전 확인(인원·명단·합계·할인/중간입회/미납·제외·기발송·채널).
  중복 방지: (기간, 원생, 청구 버전) UNIQUE + 멱등키, 재전송은 "재알림"으로 분리
- **중간 입회 계산기**: 입력 = 첫 수업일 + 반(요일·시간 자동) + 할인. 출력 = 남은 실제 수업일
  목록·회차·제외 휴무·회차별 계산·최종 청구액. **실제 수업 캘린더 기준**(단순 날짜차 금지),
  원 단위 반올림 정책 고정, 수동 수정 이유 필수, 청구 초안 저장 후 원장 확정
- **할인**: 정액+정률+회차+무료 회차+수동 조정+프로모션 코드 / 범위(원생·형제·가족·프로그램·반·
  다종목·장기·기간·이벤트) / 규칙 필드(중복 허용·최대 한도·자동/수동·차량비/등록비 포함·승인 필요).
  개인 할인 허용 조건 = 이유·전후 금액·기간·입력자·승인자·AuditLog.
  **C10-01 결합**: 할인 후 음수 금지·정률 100% 초과 금지·비정상 정액 금지·최종 상한·수동 조정 승인
- **환불 목록**: 단일 카드 → 요청 목록(상태 요약) + 접기/펼치기 상세(수강료·차량/등록비·위약금·
  회수 계산·휴무 반영·예상 환불액·규정·양측 승인·PG 상태). UI 순서 명시:
  **계산 → 원장 제안 → 학부모 승인 → 원장 최종 승인 → PG → 웹훅 COMPLETED → 재계산**
- **공지**: 모바일 = 최근 3개+읽음 비율+안 읽은 명단(재알림·대화·선택 재발송) / PC = 작성·필터·
  예약·미리보기·첨부·추적·실패 관리
- **강사**: PC 상세(기본·자격/안전·운영·권한 4군) — 민감 인사정보 모바일 비노출.
  교체 버튼은 목록 행·상세에만. 교체 flow = 기존/새 강사·반·적용일·인수인계·권한 회수·공지·미배정 확인·승인
- **설정**: 모든 행 → 실제 modal/상세. 공통 = 현재 값·설명·변경값·적용일·영향 범위·저장·되돌리기·AuditLog
