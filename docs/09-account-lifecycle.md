# 09 · 계정 라이프사이클 (F14, 리뷰#2 P0-5)

로그인 세션 · 학원 나가기 · 계정 탈퇴. 엔티티: `MembershipExitRequest` · `AccountDeletionRequest` (docs/02). 상태전이: docs/03 §6·§10.

## 1. 세션 · 로그아웃
- 로그인: 카카오(앵커)·네이버·구글·애플. 세션 = `UserSession`(기기·최근로그인).
- **로그아웃 / 모든 기기 로그아웃** — 실인증 구현 시 필수.
- 폰·이메일 변경 = 단순 저장 아님 → **OTP 재인증** + 기존 번호로 변경 알림.
- Admin·원장 계정 **MFA** 권장. 코치 퇴사(멤버십 ENDED) 시 **모든 세션·토큰 만료**(docs/03 §6).

## 2. 학원 나가기 ≠ 계정 탈퇴 (서로 다름!)
### 학원 나가기 (`MembershipExitRequest`)
소속 관계만 종료, **계정은 유지**.
```
REQUESTED → (미완료 업무 확인 · 인수인계 작성) → 원장 APPROVED
          → 담당반 권한회수 · 원생/사진/건강정보 접근차단 · 세션·토큰 만료
          → 감사기록 유지
```
- 코치는 **즉시 나가기 불가** — 반드시 요청·승인 경유("노하우는 학원에 남는다").

### 계정 탈퇴 (`AccountDeletionRequest`)
플랫폼 계정 종료. **결제·환불·출결·안전사고·감사 기록을 즉시 전삭하지 않는다.**
| 구분 | 처리 |
|---|---|
| 즉시 삭제 | 프로필(사진·bio 등) |
| 법적·운영 보관 | 결제·환불·출결·안전사고·감사 (보관기간 후 파기) |
| 익명화 | 통계·집계에 남는 참조 |
| 복구 가능 | `recoverableUntil` 기간 내 |
| 연결 해제 | 학원·자녀 관계, **자동결제 해지 확인** |

## 3. 온보딩 (리뷰#2 P1-3)
역할별 체크리스트(원장·코치·학부모) = `OnboardingChecklist`/`Step`. **하단 상시 검은 패널 X** → 홈 상단 진행카드/설정 "시작 가이드".

## 4. 서버 필수
탈퇴·나가기 전이 시 권한·세션 즉시 무효화, 보관·삭제 정책은 법률 확인 후 확정(⚠️ 학원법·개인정보보호법 — 헌법 미해결 항목).
